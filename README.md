# 🔐 Password Checker Bot

Telegram-бот для анализа надёжности паролей и генерации новых, стойких к взлому.

Проект сделан как учебный пример по теме информационной безопасности:
показывает базовые принципы оценки стойкости паролей (энтропия, наборы
символов, проверка по списку скомпрометированных паролей) и правильную
генерацию случайных значений с помощью криптографически стойкого
генератора (`secrets`, а не `random`).

## Возможности

- **Проверка пароля**: пользователь присылает боту пароль текстом, бот
  анализирует его и присылает отчёт — длину, энтропию в битах, состав
  символов, попадание в список популярных/слитых паролей и рекомендации
  по улучшению.
- **Генерация пароля**: команда `/generate [длина]` создаёт случайный
  надёжный пароль заданной длины (по умолчанию 16 символов).
- Введённые пароли **не сохраняются** и не логируются; сообщение
  пользователя с паролем сразу удаляется из чата.

## Стек

- Python 3.11+
- [aiogram 3](https://docs.aiogram.dev/) — асинхронный фреймворк для Telegram-ботов
- `secrets`, `math`, `re` — стандартная библиотека, без сторонних крипто-зависимостей
- `pytest` — тесты для логики проверки и генерации

## Установка и запуск

1. Клонируем репозиторий и ставим зависимости:

```bash
git clone https://github.com/scared110/password-checker-bot.git
cd password-checker-bot
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

2. Получаем токен бота: пишем [@BotFather](https://t.me/BotFather) в
   Telegram, отправляем `/newbot`, следуем инструкциям, копируем токен.

3. Устанавливаем токен как переменную окружения:

```bash
set BOT_TOKEN=твой_токен_сюда
```

4. Запускаем бота:

```bash
python bot.py
```

5. Пишем боту в Telegram `/start`.

## Как это работает: пара слов о логике

**Оценка энтропии.** Считаем размер алфавита символов, которые
встречаются в пароле (строчные + заглавные + цифры + спецсимволы), и
применяем формулу `entropy = length * log2(pool_size)`. Чем больше
энтропия — тем больше возможных комбинаций перебора нужно проверить
атакующему.

**Список популярных паролей.** В проекте небольшой встроенный список
для демонстрации подхода. В реальном инструменте стоит подключать
полноценные открытые датасеты слитых паролей или API
[Have I Been Pwned](https://haveibeenpwned.com/API/v3#PwnedPasswords),
который поддерживает k-anonymity — можно проверить пароль по хешу,
не отправляя его целиком на сторонний сервер.

**Генерация пароля.** Используется модуль `secrets` вместо `random`,
потому что `random` — это генератор псевдослучайных чисел, не
предназначенный для криптографии: его внутреннее состояние можно
восстановить, зная несколько выданных значений. `secrets` берёт
случайность из источника ОС и подходит для паролей, токенов, ключей.
